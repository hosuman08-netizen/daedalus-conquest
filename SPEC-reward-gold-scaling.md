# SPEC: 출석/플레이 골드 전력비례 전환 (단조+무력 동시 해결)

> CPO 트리니티 · 구현: 모피어스(COO, game.js) · 검증: 트리니티(balance-sim)
> **발단**: 군주 보상 밸런스 검토 + balance-sim — 복리 `dividendGold=legionPower×0.9` 후반 폭증 → 출석 골드(16~29일 1k 고정) **휴지 + 단조**.

## 문제 (검증됨)
- 출석 골드 16~29일 **1k 고정** = 단조 + 후반 legionPower 폭증에 **무력**(복리가 압도).
- 무과금 후반 유저에게 출석/플레이 골드가 **의미 0**.

## 해결: 비례+하한 하이브리드 (초반·후반 둘 다 유효)
- **출석 골드 = `max(기존절대값, floor(legionPower × dayMul))`**
  - `dayMul` 일차별 점증 (예: 1일 0.03 → 30일 0.30, 선형 `0.03 + day*0.009`).
  - 초반(legionPower 낮음)=절대값 하한 유지(현 보상 보존). 후반=비례(유효).
- **플레이 골드(3m~60m) = `max(절대값, floor(legionPower × playMul))`**
  - `playMul` 시간 비례 (3m 0.05 → 60m 0.5).

## 유지 (변경 X)
- **다이아·아이템·마일스톤(7일📦·15일🎁·30일🌟)** = 절대값 그대로 (다이아는 가챠화폐라 인플레 적음 — balance-sim 확인: 월 630💎=78연=SSR7~8개 후함, OK).

## 검증 (트리니티)
- balance-sim으로 legionPower 초반/중반/후반 3구간 골드 체크 → **현 보상보다 작아지지 않게**(하한 보장) + 후반 유효한지.
- 비율(dayMul/playMul) 시뮬 조정.

## 형태 고정
출석/플레이 골드 = `max(절대값, 전력×비례)` · 다이아 절대값 유지 · 점증비율은 시뮬조정.
