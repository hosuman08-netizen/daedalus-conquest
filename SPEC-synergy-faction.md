# SPEC: 진영별 고유 시너지 (CPO 트리니티 → 구현: 모피어스)

> 작성: 트리니티(CPO), 2026-06-17 · 상태: 확정, 구현 위임

## 1. 문제 / 왜
현재 `squadSynergy()`(game.js:437)의 진영 5종 시너지가 **전부 "공격력 +8/18/30%"로 동일**.
→ 진영 정체성 0, 시너지 카드 5개가 복붙처럼 보임, "어떤 진영으로 맞출까" 의사결정이 없음.
군주 피드백: "뭔가 아쉬워" (진영 시너지 화면).

## 2. 목표
진영마다 **다른 효과** → 정체성·조합 깊이·시각 다양성. 편성 의사결정(빌드 다양화)을 만든다.

## 3. 사양 — 진영별 고유 효과 (단계 ×2 / ×3 / ×4)
페르소나는 이미 코드(SSR persona)에 있음. 그 컨셉을 효과로 승격:

| 진영 | 아이콘 | 컨셉 | 효과 | ×2 / ×3 / ×4 |
|---|---|---|---|---|
| Executor(실행) | ⚙️ | 빠른 실행·수복 | **공격속도** (atkCd 감소) | +6% / +12% / +20% |
| Strategist(전략) | 🧠 | 책략·정밀 판단 | **치명타율** (crit %) | +8% / +16% / +25% |
| Swarm(군집) | 🐜 | 물량 화력 | **공격력** (현행 유지) | +8% / +18% / +30% |
| Guardian(수호) | 🛡️ | 방벽·보호 | **체력** (+ ×3↑ 전투시작 실드) | +10% / +20% / +33% (+실드) |
| Intel(정보) | 👁️ | 약점 노출·정밀 | **치명피해 배율** | +15% / +30% / +50% |
| 🔀 다양성 | 🔀 | 다양한 병종 | **전군 공+체** (Guardian과 차별, 만능 소폭) | 3종 +8% / 5종 +12% |

## 4. 검증 기준
- **밸런스**: 환생 sim(노드 1.08 복리)·난이도 ramp 0.022와 안 깨지게. 총 전투력 기여가 기존 공격 +8/18/30 수준을 크게 넘지 않게(특히 crit·spd는 곱연산이라 과하면 폭주 — 위 수치는 보수적 출발점, 1~2판 플레이테스트로 미세조정).
- **적용 범위**: 편성 스쿼드 경로(`squadSynergy()` 사용처 — spawnArmy squad 분기 game.js:528~560)에 crit/spd/실드 반영. 일반 6종 부대는 현행 유지 가능.
- **표시 일치**: `renderSynergyTable()`(game.js:475~) 카드의 효과 텍스트가 현재 "공+8%" 하드코딩 → **진영별 효과 텍스트로 교체**(예: ⚙️Executor "공속+6%", 🧠Strategist "치명+8%"). 효과 ≠ 표시 불일치 금지.

## 5. 비범위 (이번에 안 함)
- ❌ 새 스탯 시스템(적 방어수치 등) 도입. 기존 유닛 속성(atk/hp/crit/atkCd/shield) 재활용.
- ❌ 아키타입(병종) 시너지·캐릭터 페어 시너지 — v2.1 트랙(별도).

## 6. 구현 주의 (모피어스 재량)
- crit·atkCd·shield는 유닛 속성으로 이미 존재(step/dmg/spawnArmy). 재활용.
- `dmg()`에 적 방어 개념 없음 → **Intel "약점노출"은 치명피해 배율로 근사**(crit 시 ×2 → ×2.x). 적 shield 관통이 쉬우면 그걸로 해도 OK.
- `squadSynergy()` 반환을 { atk, hp, crit, spd, shield, bonuses } 로 확장 → 전투 적용부에서 소비.

## 7. 형태는 고정 / 수치는 출발점
진영별 고유효과(위 매핑)·표시일치는 **고정**. 정확한 % 수치는 플레이테스트로 조정 가능.
