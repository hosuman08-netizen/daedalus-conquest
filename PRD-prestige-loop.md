# PRD: 메타 환생 루프 (Prestige / Ascension)

> 작성: 2026-06-16 · 작성자: CPO(베라) · 상태: 초안

## 1. 문제 / 왜 지금
- **고통**: 신규 유저가 ~7일차에 ch21+ 지수 난이도 벽(`enemyPowerMul` game.js:634)에 막히고, 캠페인·무한탑·일일이 모두 같은 엔진이라 "할 새 것"이 없음. 코드상 엔드게임 목표 0개.
- **안 풀면**: D7 이후 대량 이탈 → 리텐션 붕괴 → 수익화를 얹을 잔존 유저 자체가 없음(수익화는 잔존 위에만 선다).
- **왜 지금**: 출시 막는 두 벽 중 ①콘텐츠 절벽이 1순위. 아레나·길드·배틀패스는 코드 0%·몇 주짜리 → 너무 무겁다. 환생은 **기존 캠페인 엔진을 재활용**해 가장 싸게 무한 루프를 만든다.

## 2. 목표 (성공 기준 — 측정 가능)
- **1차 지표**: D7 리텐션, 그리고 "벽에서 막힌 유저"의 다음날 복귀율 ↑ (현재 stuck ~30%, sim 기준).
- **보조 지표**: 1인당 환생 횟수(루프가 돌면 ≥1), D14 도달 유저 비율, 세션당 플레이 시간 유지.
- **가드레일**: 1회차(첫 환생 전) 플레이타임이 짧아지면 안 됨 / 환생이 *강제*처럼 느껴지면 안 됨(자발적 선택) / 기존 진행 손실로 인한 불만 toast/이탈 스파이크 없을 것.

## 4. 범위 (이번에 만드는 것)
- [ ] **환생 액션**: 일정 챕터(예: ch15+) 도달 시 "환생" 버튼 활성화 → 진행도(챕터·골드 등 휘발 자원) 리셋, 영구 자원 **에테르(Ether)** 획득. 획득량 = 도달 챕터 기반(높이 갈수록 ↑).
- [ ] **영구 배율**: `META.prestige`(현재 장식용, game.js:251-259)를 **실제 영구 전투 배율**로 승격 — 에테르 누적 → 아군 공/체 영구 +%.
- [ ] **에테르 상점(최소)**: 에테르로 사는 영구 업그레이드 3~5개(예: 시작 골드, 가챠 할인, AFK 레이트, 공격력, 첫 배치 수). 노드 몇 개로 시작.
- [ ] **재정복 가속 체감**: 환생 후 초반 챕터가 *눈에 띄게* 빨라지게(영구 배율 적용 확인) — "더 멀리 갈 수 있다" 도파민.
- [ ] **난이도 곡선 소폭 완화**: `enemyPowerMul` ch21+ 계수 0.028 → 환생 루프와 균형 맞게 재튜닝(balance-sim.js로 검증).
- [ ] **명확한 UI 피드백**: 환생 확인 모달(얻는 것/잃는 것 명시), 에테르·영구배율 표시, "이번엔 어디까지?" 목표 제시.

- **알려진 리스크**:
  - 첫 환생 진입점이 너무 늦으면(ch30+) 벽에서 이미 이탈 → 진입을 벽 *직전*(ch15~18)에 둔다.

  - 리셋 불만 → 모달에서 "얻는 것"을 크게, "환생은 자발적"임을 분명히.

## 📊 확정 수치 시트 (sim 검증 — `node balance-sim-prestige.js`)

**핵심 발견(위험 가정 검증 결과)**: 영구배율은 반드시 **복리(곱연산)**. 적 파워가 +5챕마다 ~1.4배로 오르므로 **선형 가산 배율은 무조건 stall** → 환생이 무의미해짐. 복리로 짜면 매 환생 +4~7챕 전진하며 무한 루프가 산다. 직전 벽 재돌파는 항상 '즉시'(영구배율이 초반을 녹임 = 도파민 ✓).

**확정 상수**:
- **진입 게이트**: ch18 도달 시 환생 버튼 활성.
- **에테르 획득** = `round(20 × 1.18^((도달챕터−18)/2))` ≈ 적 파워/12에 비례. (예: ch18→20, ch25→31, ch30→51, ch40→102. 깊이 갈수록 기하급수.)
- **영구배율: 노드 1레벨당 해당 스탯 +8% 복리 (1.08).** ⚠️ **교정 2026-06-16(모피어스 빌드 재sim)**: 본문 초안의 "+18%/lv(1.18)"은 *단일 풀(pool) M* 가정값. 실제 게임은 ATK·HP **분리노드**라 전투력 = atk×hp = `1.08^(공+체)` → 분리모델에 1.18 적용 시 **5환생째 ch300 천장 폭주**(재현: 세션 `/tmp/sim-game-model.js`). **빌드 채택 = 1.08.** 형태(복리)·진입ch18·3노드는 그대로.
  - 「공세(Might)」: 아군 전체 ATK +8%/lv (복리)
  - 「불굴(Bulwark)」: 아군 전체 HP +8%/lv (복리)
  - 「쇄도(Momentum)」: 시작 골드 +300/lv · 골드획득 +18%/lv (재돌파 *속도* — 전투력 아님 → 폭주 무관)
- **에테르 획득** = `round(20 × 1.18^((도달챕터−18)/2))` — 이 1.18은 *획득 곡선*(깊이비례)이라 파워와 무관, 유지. (재sim상 1.08 노드와 건강하게 맞물림.)
- **노드 비용** = `round(5 × 1.25^lv)` 에테르 (lv0→5, 1→6, 2→8, 3→10, 4→12, 5→15…). 노드별 독립 레벨.
- **난이도**: `enemyPowerMul` ch21+ 계수 **0.028 → 0.022** 소폭 완화(루프 균형).

**검증된 궤적(빌드 재sim, 진입ch18·노드+8%)**: 환생 1~8회차 도달벽 = 18→23→27→29→31→35→37→40, 매회 +2~5 전진, 폭주 없이 무한 생존✓. (초안의 18→25→…→55은 폐기된 풀모델 수치.)

**컷라인(D-5 출시)**: 위 **3노드(공세·불굴·쇄도)면 루프 완성** — 4·5번째 노드(현신/정수)는 출시 후. 곱연산 배율 + 깊이비례 에테르 두 가지가 핵심, 이것만 지키면 산다.

**주의**: sim은 *상대 균형* 모델(절대값 근사). 위 상수는 출발점 — 실제 빌드에서 1~2판 플레이테스트로 미세조정(특히 노드별 비용 분배). 형태(복리·진입ch18·3노드)는 고정.
